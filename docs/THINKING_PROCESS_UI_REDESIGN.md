# 思考プロセス UI 全面改善設計書

**作成日**: 2026-05-27  
**目的**: 「なぜこの提案か」画面を全面的に改善し、直感的な理解を優先したUIに変更

---

## 🎯 目標

従来の Knowledge Graph 型の複雑な可視化から、**5ステップの線形思考プロセス表示**に変更し、以下を実現する：

1. **一瞬で理解できる**: 「なぜこの車なのか」が直感的にわかる
2. **販売店でも説明しやすい**: 営業担当者が顧客に説明する際に使えるレベル
3. **本部長クラスにも納得感**: 技術説明ではなく、ビジネス価値が伝わる

---

## 📐 新レイアウト: 5ステップ構造

### 横方向フロー

```
Step 1          Step 2          Step 3          Step 4          Step 5
───────────    ───────────    ───────────    ───────────    ───────────
あなたの価値観  →  検出された負荷  →  必要な体験  →  必要な機能  →  おすすめ車種
```

### 各ステップの内容

#### Step 1: あなたの価値観
- 背景色: 青系 (`bg-blue-50`, `border-blue-500`)
- 内容: Quick Questions から抽出された上位3つの価値軸
- 表示例:
  ```
  ✓ 効率重視
  ✓ 楽しさも欲しい
  ✓ 日常利用中心
  ```

#### Step 2: 検出された負荷
- 背景色: 赤系 (`bg-red-50`, `border-red-500`)
- 内容: 移動時のストレス・不安（最大3件）
- 表示例:
  ```
  ✓ 維持費への不安
  ✓ 渋滞ストレス
  ```

#### Step 3: 必要な体験
- 背景色: 緑系 (`bg-green-50`, `border-green-500`)
- 内容: **負荷を解決するための体験**（機能ではなく体験レベル）
- 表示例:
  ```
  ✓ 維持費を気にしすぎない
  ✓ 渋滞でも疲れにくい
  ✓ 毎日の移動が快適
  ```

#### Step 4: 必要な機能
- 背景色: 紫系 (`bg-purple-50`, `border-purple-500`)
- 内容: **体験を実現するための具体的機能 + 理由**
- 表示例:
  ```
  Google Map統合
  → 渋滞ストレス軽減

  Google Assistant
  → 運転中操作負荷軽減

  Google Play対応
  → 日常体験の快適性向上
  ```

#### Step 5: おすすめ車種
- 背景色: ゴールド系 (`bg-[#FFFAF0]`, `border-[#B8920C]`)
- 内容: 推薦車種 + マッチスコア + 理由
- サイズ: やや大きめ（280px）
- 表示例:
  ```
  CR-V e:HEV
  
  マッチスコア: 92%
  
  理由:
  日常利用 × 渋滞ストレス × 維持費不安
  との相性が高いため
  ```

## 🎨 デザイン原則

### Apple × Automotive × Explainable AI

1. **余白を増やす**: 情報を詰め込みすぎない
2. **カードを大きく**: 各ステップをゆとりあるカードで表現
3. **情報量を減らす**: 最大3項目まで
4. **一瞬で理解**: テキストは簡潔に、視覚的階層を明確に

### カラーパレット

| Step | 背景色 | ボーダー色 | 意味 |
|------|--------|------------|------|
| 1 | `bg-blue-50` | `border-blue-500` | 価値観（信頼感） |
| 2 | `bg-red-50` | `border-red-500` | 負荷（アラート） |
| 3 | `bg-green-50` | `border-green-500` | 体験（ポジティブ） |
| 4 | `bg-purple-50` | `border-purple-500` | 機能（テクノロジー） |
| 5 | `bg-[#FFFAF0]` | `border-[#B8920C]` | 推薦（プレミアム） |

## 🔄 アニメーション仕様

### 段階的表示

- 各ステップを左から右へ順番に表示（800ms間隔）
- 表示時は `translate-y-4` から `translate-y-0` にフェードイン
- 矢印（→）は両隣のステップが表示された後に表示

### スキップ機能

- 「アニメーションをスキップ」ボタンで即座に全表示
- `prefers-reduced-motion` 対応（モーション低減設定時は即座に全表示）

## 📍 右側パネル: AIの判断理由

### 従来の問題

- 「推薦ロジック」が数式っぽい（例: `効率 × 楽しさ × 維持費 → Google Play...`）
- 本部長クラスには抽象的すぎる

### 改善後

#### 1. 重視価値（バーグラフ）

既存のバーグラフを維持、説明を追加：

```
効率 100%
→ 日々の移動負荷を減らしたい傾向

楽しさ 49%
→ 移動体験の質も重視

安心 23%
→ 安全性は最低限重視
```

#### 2. AIの判断理由（自然言語化）

数式表記を廃止し、自然な日本語に：

```
あなたは
「効率」と「楽しさ」を重視しながら、

「維持費」と
「渋滞ストレス」に負担を感じる傾向があります。

そのため、

日常使いしやすく、
維持費バランスが良く、

渋滞時のストレスを軽減しやすい
CR-V e:HEVをおすすめしています。
```

#### 3. 検出された負荷

赤いアイコン（`!`）でアラート感を出す：

```
! 維持費への不安
! 渋滞ストレス
```

## 📝 技術メモ

### Tailwind CSS クラス

カード基本構造:
```tsx
className="rounded-lg border-2 bg-white shadow-sm transition-all duration-500"
```

アニメーション:
```tsx
className={visible ? "translate-y-0 opacity-100" : "translate-y-4 opacity-0"}
```

### 横スクロール対応

```tsx
<div className="relative w-full overflow-x-auto pb-8">
  <div className="flex min-w-[1200px] items-start justify-between gap-6 px-4">
    {/* Steps */}
  </div>
</div>
```
